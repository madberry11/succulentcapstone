Description

My website was based on the Freelancer Bootstrap Theme. The links to all free materials I used are included in the copyrights.html page, so if you want you can look at the original to see how it has been changed. I chose this theme because it matched very closely with what I had in mind, so I did not change much about how it looks, just the contents. The website has four 'pages' but all included on the same page as sections. The copyrights.html is not really a part of the website content, I just added it to be safe about Creative Commons materials.

The theme of the website is succulents. I took up this hobby last year, and it is pretty much my only interest that I have high quality photos about. I would have wanted to write in a lot more detail about care and propagation, but I did not want to make the website too long and boring for the graders. If you are actually interested in the topic, I suggest taking a look at the links in the bottom.

Styling

The look of the template was very similar to what I wanted, so I did not change it too much. In addition, many of the elements I used require a specific CSS file to work. My own stylesheet is style.css. It mostly just contains rules to override the original styling from the various CSS files. Working with so many different stylesheets, it was quite difficult to pinpoint exactly which file and which rule was causing problems when something did not look the way it was supposed to. I had to regularly use the Inspector in the Web Developer tools to see what rules apply to the elements I was trying to style.

Responsiveness

A lot of the responsiveness of the website comes from the original template, including

  • minimised menu with hamburger icon on small screens,
  • responsive image sizes,
  • 'scroll to top' link on small screens.

In addition to these, I changed the font sizes to be responsive to screen size using rem measurements, and in some cases I set different rem font size for the same element for different resolutions. I personalised the minimised menu so that each line is tall enough to be easily clicked on a mobile device.

In the Care section originally I had the tabbed layout, but once I hosted a website and I could access it from my phone I could see that the layout does not work well for mobile phones because you need to keep scrolling up to click on the link to the next topic. I added the Accordion as a special consideration for small screen sizes but I kept the tabbed look for wide screens.

I have tested the website on Android smartphone, Android tablet and Windows laptop. These are all the devices I have, I apologise if it does not look quite as nice on other devices.

Validation and Accessibility

Both index.html and copyrights.html validate without any errors or warnings in the W3C validator:

Screenshot of index.html validating

Screenshot of copyrights.html validating

For the WAVE validation I had four errors and two alerts. The alerts I could correct (1x missing first level heading and 1x suspicious link text). The errors were 1x missing first level heading and 3x empty link. I went over and over the source code and could not find what was causing these errors. They were not visibly flagged on the WAVE page either. From the number of errors I got to the conclusion that they must be caused by the backward, forward and exit links from the Lightbox. If I had more time to go over the code of the Lightbox in detail, it probably would have been possible to fix these.

I used the WebAim Colour Contrast Checker and as a result had to give up on my pretty colour transition background and use a much darker green instead.
